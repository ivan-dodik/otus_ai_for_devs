# Домашнее задание "Минимальный агент на LangChain"

## Цель

Собрать и запустить минимального AI-агента на LangChain, который выступает в роли натурально-языковой обёртки над API.

## Описание/Пошаговая инструкция выполнения домашнего задания

Агент должен:

* принимать запрос пользователя на естественном языке;
* интерпретировать намерение пользователя;
* вызывать один или несколько API-методов через tool;
* возвращать структурированный результат с указанием статуса выполнения.

**Результат:** репозиторий/PR с рабочим LangChain-агентом + короткий отчёт и список используемых промптов.

В результате выполнения этого задания у вас появляется небольшой проект (или папка в существующем проекте) с агентом, который позволяет работать с API через естественный язык, а не напрямую через SDK или HTTP.

## Что нужно сделать

### 1. Подготовьте окружение

* Создайте минимальный проект или отдельную папку в существующем репозитории.
* Зафиксируйте зависимости (`requirements.txt` или `pyproject.toml`).

### 2. Выберите LLM-провайдера

* Облачный (OpenAI / Anthropic и т.п.) или локальный (Ollama, LM Studio и др.).
* Секреты и ключи не коммитить.
* В репозитории должен быть `.env.example` и описание настройки.

### 3. Определите API-домен

Выберите один из вариантов:

* mock API (локальный HTTP-сервис);
* публичное тестовое API;
* внутренний API вашего проекта;
* минимальный API, реализованный прямо в этом же репозитории.

Достаточно 2–4 операций, например:

* создать сущность;
* получить сущность;
* обновить состояние;
* получить список / статистику.

### 4. Реализуйте tool для вызова API

* Tool должен быть оформлен как LangChain tool.
* Внутри tool — явный HTTP-вызов или функция-обёртка.
* Tool возвращает структурированный результат (JSON / dict → string).

### 5. Соберите агента

* Настройте системный prompt:
  * роль агента (API-оператор);
  * ограничения (что он может и не может делать);
  * правила вызова tool.
* Подключите API-tool.
* Реализуйте запуск из CLI, например:

  ```bash
  python main.py "создай пользователя с именем Alex"
  ```

### 6. Определите контракт ответа

Ответ агента должен быть в фиксированном формате, например:

```text
Status: success | error
Action: <описание действия>
Data: <результат API>
Errors: <если есть>
```

### 7. Проверка (обязательный шаг)

* Выполните минимум 5 разных пользовательских запросов.
* Минимум 3 запроса должны приводить к реальному вызову API-tool.
* Зафиксируйте запросы и ответы.

### 8. Сохраните все используемые промпты

* Оформите список промптов в Markdown.
* Включите системный prompt и пользовательские шаблоны.

## Формат сдачи

* Ссылка на PR / репозиторий / ветку.
* Короткий отчёт (`report.md` или `README`), где указано:
  * какой LLM используется и как его настроить;
  * какое API выбрано и какие операции поддерживаются;
  * как запустить агента (команды);
  * 5 тестовых запросов и полученные результаты.
* Раздел **«Использованные промпты»** в формате Markdown.

## Критерии оценки

Работа считается выполненной, если:

1. **Агент на LangChain запускается** по инструкции из репозитория.
2. **Реализован минимум один API-tool с реальным вызовом.**  
   *Подтверждение:* укажите файл и диапазон строк, где объявлен tool и выполняется реальный вызов API (например, `tools/api_tool.py:L12–L48`). Также добавьте вывод результата tool в консоль для дебага (например, `print()`/логгер) и укажите, где именно это сделано (файл/строки).
3. **Агент корректно интерпретирует запросы и выбирает нужный API-метод.**  
   *Подтверждение:* приведите минимум 1 пример запроса, который должен привести к вызову конкретного API-метода, и укажите ожидаемый метод (например: запрос → `createUser`). Желательно приложить фактический фрагмент лога/trace, подтверждающий вызов tool.
4. **Ответ агента соответствует заданному контракту.**  
   *Подтверждение:* укажите, где в репозитории описаны требования к формату ответа агента (например, `README.md`: раздел “Контракт ответа” или `prompts/system.md:L1–L30`).
5. **Приведены минимум 5 проверочных запросов.**
6. **Промпты оформлены и приложены.**
7. **Секреты не закоммичены.**
```

Я нашёл интересное API - https://kitsu.docs.apiary.io/ - большой список аниме с тегами и другой информацией.

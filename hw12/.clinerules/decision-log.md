## 2026-05-23 — Выбор API: Kitsu вместо Jikan

**Контекст:** Нужно публичное API для получения данных об аниме без аутентификации.

**Рассматриваемые варианты:**
- **Jikan (MyAnimeList):** Требует API key, более строгие лимиты
- **Kitsu:** Полностью публичное, не требует ключа, богатая документация

**Решение:** Kitsu API

**Обоснование:** Отсутствие необходимости в аутентификации упрощает настройку для учебного проекта. Kitsu предоставляет все необходимые эндпоинты: поиск, детали, жанры, тренды.

**Последствия:** Все 4 инструмента используют `requests.get()` без заголовков авторизации.

## 2026-05-23 — Выбор LLM: Ollama + llama3.1:8b

**Контекст:** Требуется локальный LLM с поддержкой function calling.

**Рассматриваемые варианты:**
- **OpenAI API:** Требует ключ и интернет, не подходит для локального запуска
- **Ollama + llama3.1:8b:** Локально, поддержка tool calling, хорошая мультиязычность
- **Ollama + mistral:7b:** Легче, но хуже с русским языком

**Решение:** Ollama + llama3.1:8b (по умолчанию, конфигурируется)

**Обоснование:** llama3.1:8b имеет хорошую поддержку русского языка и function calling, что критично для корректной работы агента.

**Последствия:** Модель можно переопределить через `--model` флаг или `OLLAMA_MODEL` в `.env`.

## 2026-05-23 — get_anime_info: композитный инструмент вместо цепочки

**Контекст:** LLM не справлялась с последовательным вызовом `search_anime` → `get_anime_details`.

**Рассматриваемые варианты:**
- **A:** Новый инструмент `get_anime_info(name)` — поиск + детали внутри одной функции
- **B:** Только улучшение промпта (цепочка инструментов)
- **C:** Переименовать `get_anime_details` → `get_anime_info(query)`

**Решение:** Вариант A — новый композитный инструмент

**Обоснование:** LLM часто ошибается при каскадных вызовах. Один инструмент надёжнее и быстрее.

**Последствия:** В `tools/kitsu_tools.py` не добавлялся новый инструмент (реализовано через улучшение промпта для цепочки search_anime → get_anime_details в agent/prompts.py). Позже перешли на вариант A — добавлен get_anime_info.

## 2026-05-23 — Python logging вместо print() для дебага

**Контекст:** `print()` для отладки загрязняет вывод и не управляется флагами.

**Рассматриваемые варианты:**
- **Флаг `--verbose`:** Добавить парсинг аргументов, оставить `print()`
- **Python logging + `--debug`:** Заменить на `logger.debug()`, управлять через CLI

**Решение:** Python logging с `--debug` флагом

**Обоснование:** Стандартный подход, `logger.debug()` не выводится по умолчанию, можно включать для Flask UI через `.env`.

**Последствия:** Все `print()` заменены на `logger.debug()`. Добавлен `logging.basicConfig()` с уровнем из `config.DEBUG`.

## 2026-05-23 — Перевод описаний через LLM вместо внешнего API

**Контекст:** Агент должен отвечать на языке пользователя, включая перевод synopsis/description.

**Рассматриваемые варианты:**
- **Внешний API перевода:** Google Translate, DeepL — зависимость от интернета
- **LLM сам переводит:** Усилить system prompt инструкцией перевода

**Решение:** Перевод через LLM (усиление промпта)

**Обоснование:** Не требует внешних зависимостей, llama3.1:8b достаточно хорошо переводит. Названия/жанры/теги остаются в оригинале.

**Последствия:** В system prompt добавлена инструкция: "обязательно переведи synopsis, description на язык пользователя. Названия аниме, жанры, теги можно оставить в оригинале."

## 2026-05-24 — filter[categories] вместо filter[genres] для поиска похожих аниме

**Контекст:** При запросе "Найди аниме похожее на..." для разных аниме возвращался **одинаковый** список рекомендаций. Инструмент `find_similar_anime` использовал `filter[genres]=bounty-hunter` и другие category slugs, но Kitsu API игнорировало эти фильтры.

**Рассматриваемые варианты:**
- **Вариант A:** Использовать `filter[genres]` с proper genre slugs (action, comedy, etc.) — но это даёт слишком общие результаты
- **Вариант B:** Использовать `filter[categories]` с category slugs (bounty-hunter, space, etc.) — более специфичные результаты
- **Вариант C:** Использовать оба фильтра одновременно — максимальная точность, но больше API вызовов

**Решение:** Вариант B — использовать `filter[categories]` вместо `filter[genres]`

**Обоснование:** 
- Kitsu API различает genres (основные жанры: action, comedy, drama) и categories (специфичные теги: bounty-hunter, space, magical-girl)
- Функция `_get_anime_genres()` возвращает category slugs из эндпоинта `/anime/{id}/categories`
- При использовании `filter[genres]=bounty-hunter` API игнорирует фильтр (bounty-hunter не является валидным genre) и возвращает дефолтный список популярных аниме
- `filter[categories]=bounty-hunter` корректно фильтрует по специфичным тегам

**Последствия:**
- В `tools/kitsu_tools.py` изменено: `filter[genres]={slug}` → `filter[categories]={slug}` в функции `find_similar_anime`
- Теперь рекомендации действительно различаются для разных аниме
- Cowboy Bebop (bounty-hunter, space) → Trigun, Black Lagoon, Ghost in the Shell
- Sailor Moon (magical-girl, magic) → Chainsaw Man, Sousou no Frieren, Tongari Boushi no Atelier
- Naruto (ninja, martial-arts) → Jujutsu Kaisen, Attack on Titan, Hunter x Hunter

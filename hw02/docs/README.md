# Документация приложения "Мини-анкета"

## 📚 О документации

Эта документация предназначена для разработчиков с базовыми знаниями Python и HTML, которые хотят понять архитектуру и принципы работы full-stack веб-приложения "Мини-анкета".

**Целевая аудитория:**
- Начинающие разработчики Python
- Студенты, изучающие веб-разработку
- Разработчики, желающие понять принципы создания REST API
- Все, кто хочет разобраться во взаимодействии frontend и backend

## 📖 Структура документации

### 1. [Архитектура приложения](ARCHITECTURE.md)

**Что вы узнаете:**
- Общий обзор системы и её компонентов
- Архитектурные решения и их обоснование
- Взаимодействие между frontend и backend
- Поток данных в приложении
- Технологический стек и причины выбора
- Вопросы безопасности
- Масштабируемость и ограничения

**Время изучения:** 1-2 часа

### 2. [Как работает Backend](BACKEND_GUIDE.md)

**Что вы узнаете:**
- Структура Flask приложения
- Инициализация и конфигурация
- Настройка CORS для跨доменных запросов
- Создание REST API endpoint'ов
- Валидация входящих данных
- Rate limiting для защиты от злоупотреблений
- Хранение данных в памяти
- Обработка ошибок и исключений
- Логирование событий
- Запуск и отладка backend

**Время изучения:** 1-2 часа

### 3. [Как работает Frontend](FRONTEND_GUIDE.md)

**Что вы узнаете:**
- HTML структура страницы
- CSS стилизация и адаптивный дизайн
- JavaScript логика и управление состоянием
- Асинхронные запросы к API
- Манипуляции с DOM
- Обработка ошибок на клиенте
- Оптимизация производительности
- Отладка и тестирование frontend

**Время изучения:** 1-2 часа

### 4. [Практические примеры и упражнения](PRACTICE.md)

**Что вы узнаете:**
- Базовые примеры кода для начинающих
- Практические упражнения для закрепления материала
- Расширение функциональности приложения
- Частые ошибки и способы их решения
- Дополнительные ресурсы for изучения

**Время изучения:** 2-4 часа (с выполнением упражнений)

## 🎯 Рекомендуемый порядок изучения

1. **Начните с [Архитектуры](ARCHITECTURE.md)** - получите общее представление

3. **Перейдите к [Frontend](FRONTEND_GUIDE.md)** - изучите клиентскую часть
4. **Выполните [Упражнения](PRACTICE.md)** - закрепите знания на практике

### Для опытных разработчиков:

- Пропустите общие разделы
- Сосредоточьтесь на конкретных интересующих темах
- Используйте документацию как справочник

# Руководство по быстрому старту

## 🚀 Запуск приложения

### 🎯 Быстрый запуск (рекомендуется)

**Одной командой из корневой директории:**

```bash
# Для Linux/macOS
./start.sh

# Или кроссплатформенный вариант
python start.py
```

Этот скрипт автоматически:
- Проверит наличие Python и зависимостей
- Установит все необходимые зависимости
- Запустит backend сервер на порту 5000
- Запустит frontend сервер на порту 8080
- Предоставит ссылки на все эндпоинты

**После запуска вы увидите:**
```
📱 Frontend:        http://localhost:8080
🔧 Backend:         http://localhost:5000
📊 Swagger UI:      http://localhost:5000/apidocs
📝 Answers page:    http://localhost:8080/answers.html
```

Для остановки нажмите `Ctrl+C`.

### 📝 Пошаговый запуск (альтернативный)

Если вы предпочитаете запускать компоненты отдельно:

#### 1. Запуск Backend

Откройте терминал и выполните:

```bash
# Перейдите в директорию проекта
cd /home/ai/aidev/hw02

# Установите зависимости (первый запуск) - автоматически создаст venv
(cd backend && uv sync)

# Запустите сервер backend
uv run python backend/app.py
```

Или, если вы предпочитаете использовать виртуальное окружение:

```bash
# Создайте и активируйте виртуальное окружение (первый запуск)
uv venv
source .venv/bin/activate

# Установите зависимости (первый запуск)
uv pip install -e backend/

# Запустите сервер backend
python backend/app.py
```

Сервер запустится на `http://localhost:5000`.

#### 2. Открыть Frontend

**Вариант A: Прямое открытие файла**
- Откройте `frontend/index.html` в вашем веб-браузере

**Вариант B: Использование локального сервера (рекомендуется)**
```bash
# В новом терминале (backend должен быть запущен)
cd frontend
python -m http.server 8080
```
Затем откройте `http://localhost:8080` в вашем браузере.


Перезапустите backend после внесения изменений.

## 🔧 Решение проблем

### Backend не запускается
- Проверьте, установлен ли Python 3.8+: `python3 --version`
- Убедитесь, что вы в директории проекта
- Проверьте, не занят ли порт 5000: `lsof -i :5000` или `netstat -tulpn | grep 5000`

### Frontend не подключается
- Убедитесь, что backend запущен: `curl http://localhost:5000/health`
- Проверьте, что frontend использует правильный URL API
- Попробуйте использовать локальный HTTP сервер вместо прямого доступа к файлу

### Ошибки CORS
- В backend включена поддержка CORS
- Используйте локальный HTTP сервер для frontend (см. Шаг 2, Вариант B)

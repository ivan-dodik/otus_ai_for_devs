# Приложение "Мини-анкета"

Простое full-stack веб-приложение, состоящее из Python Flask backend и vanilla HTML/JavaScript frontend.

## 📋 Структура проекта

```
hw02/
├── backend/
│   ├── app.py              # Flask backend приложение
│   ├── pyproject.toml      # Зависимости Python
│   └── README.md          # Специфичные для backend инструкции
├── frontend/
│   ├── index.html         # Основная HTML страница
│   └── script.js          # JavaScript логика frontend
├── test_api.sh            # Скрипт тестирования API
└── README.md              # Этот файл
```

## 🚀 Быстрый старт

### Требования
- Python 3.8 или выше
- uv (менеджер пакетов Python) - [установка здесь](https://docs.astral.sh/uv/getting-started/installation/)
- Веб-браузер

### Шаг 1: Установка зависимостей backend

Откройте терминал и перейдите в корневую директорию проекта:

```bash
# Из корневой директории проекта (автоматически создаст venv)
(cd backend && uv sync)
```

Или, если вы хотите установить зависимости вручную:

```bash
cd backend
uv pip install -e .
```

Или, если вы хотите установить только зависимости без установки пакета:

```bash
cd backend
uv pip install -r pyproject.toml
```

### Шаг 2: Запуск сервера backend

```bash
uv run python app.py
```

Или, если вы установили зависимости в виртуальное окружение:

```bash
python app.py
```

Сервер запустится на `http://localhost:5000`. Вы должны увидеть вывод like:

```
Запуск Mini-Questionnaire Backend...
Доступные эндпоинты:
  GET  /questions - Получить список вопросов
  POST /answers   - Отправить ответы
  GET  /answers   - Просмотр сохраненных ответов (отладка)
  GET  /health    - Проверка работоспособности
  GET  /apidocs   - Swagger документация API
 * Running on http://0.0.0.0:5000
```

### Шаг 3: Открыть Frontend

У вас есть два варианта:

**Вариант A: Прямое открытие файла**
- Просто откройте `frontend/index.html` в вашем веб-браузере (дважды щелкните файл или используйте "Open File" в браузере)

**Вариант B: Использование локального сервера (рекомендуется)**
Если вы столкнетесь с проблемами CORS при прямом доступе к файлу, используйте простой HTTP сервер:

```bash
# Из корневой директории проекта
cd frontend
python -m http.server 8080
```

Затем откройте `http://localhost:8080` в вашем браузере.

### Шаг 4: Тестирование приложения

1. Frontend автоматически загрузит вопросы из backend
2. Заполните ваши ответы
3. Нажмите "Отправить ответы"
4. Вы должны увидеть сообщение "Спасибо!"

## 🔧 Эндпоинты API

### GET /questions
Возвращает список вопросов анкеты.

**Ответ:**
```json
[
  {"id": 1, "question": "Как вас зовут?", "type": "text"},
  {"id": 2, "question": "Сколько вам лет?", "type": "number"},
  ...
]
```

### POST /answers
Отправляет ответы пользователя.

**Тело запроса:**
```json
{
  "question-1": "Иван Иванов",
  "question-2": "25",
  ...
}
```

**Ответ:**
```json
{
  "message": "Спасибо! Ваши ответы записаны."
}
```

### GET /answers
Возвращает все сохраненные ответы (для отладки/тестирования).

### GET /health
Эндпоинт проверки работоспособности.

### GET /apidocs
Swagger UI для интерактивной документации API.

## 🧪 Тестирование API

Вы можете протестировать API с помощью предоставленных скриптов:

**Bash скрипт (Linux/macOS):**
```bash
bash test_api.sh
```

**Python скрипт (Кроссплатформенный):**
```bash
# Убедитесь, что виртуальное окружение активировано
uv run python test_api.py
```

Или вручную с curl:

```bash
# Проверка работоспособности
curl http://localhost:5000/health

# Получить вопросы
curl http://localhost:5000/questions

# Отправить ответы
curl -X POST http://localhost:5000/answers \
  -H "Content-Type: application/json" \
  -d '{"question-1": "Алиса", "question-2": "30", "question-3": "Python", "question-4": "5", "question-5": "VS Code"}'

# Просмотр сохраненных ответов
curl http://localhost:5000/answers
```

## 🔧 Решение проблем

### Backend не запускается
- Убедитесь, что Python 3.8+ установлен: `python --version`
- Убедитесь, что зависимости установлены: `uv pip install -e backend/`
- Проверьте, не занят ли порт 5000: `lsof -i :5000` или `netstat -tulpn | grep 5000`

### Frontend не подключается к backend
- Убедитесь, что backend запущен on `http://localhost:5000`
- Проверьте консоль браузера на наличие ошибок
- Попробуйте использовать локальный HTTP сервер для frontend вместо прямого доступа к файлу

### Ошибки CORS
- В backend включена поддержка CORS, но некоторые браузеры ограничивают прямой доступ к файлам
- Используйте локальный HTTP сервер для frontend (см. Шаг 3, Вариант B)

## 📝 Настройка

### Изменение вопросов
Отредактируйте список `QUESTIONS` in `backend/app.py`:

```python
QUESTIONS = [
    {"id": 1, "question": "Ваш вопрос здесь", "type": "text"},
    # Добавьте больше вопросов...
]
```

### Изменение порта backend
Отредактируйте последнюю строку in `backend/app.py`:

```python
app.run(debug=True, host='0.0.0.0', port=5001)  # Измените порт на 5001
```

Затем обновите `API_URL` in `frontend/script.js`:

```javascript
const API_URL = 'http://localhost:5001';
```

## 📊 Хранение данных

Ответы хранятся в памяти (в списке Python) и будут потеряны при перезапуске backend сервера. Для постоянного хранения вам потребуется интегрировать базу данных.

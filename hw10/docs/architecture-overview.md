# Архитектурный обзор Betaflight

## 1. Назначение

Betaflight — прошивка для полётных контроллеров мультироторных и самолётных БПЛА. Основной фокус: производительность полёта, передовые функции, широкая поддержка железа. Де-факто стандарт в сообществе FPV-дронов.

---

## 2. Ключевые директории

```
betaflight/
├── src/main/          # Исходный код прошивки (C)
│   ├── fc/            # Flight Controller — ядро: инициализация, главный цикл, задачи
│   ├── flight/        # PID-контроллеры, микшер, режимы полёта
│   ├── drivers/       # HAL — драйверы периферии (SPI, I2C, UART, ADC, таймеры, DMA и т.д.)
│   ├── sensors/       # Чтение гироскопа, акселерометра, барометра, компаса, GPS
│   ├── rx/            # Приёмники: CRSF, SBUS, IBUS, DSM, SUMD, PPM, PWM
│   ├── telemetry/     # CRSF, SmartPort, HoTT, MAVLink, IBUS
│   ├── blackbox/      # Логирование полётных данных
│   ├── osd/           # Наложение видео (OSD) через MAX7456
│   ├── cms/           # On-screen меню для настройки в поле
│   ├── msp/           # Multwii Serial Protocol — связь с конфигуратором
│   ├── cli/           # Терминальный интерфейс настройки
│   ├── io/            # LED-лента, VTX, транспондер, пьезо-пищалка
│   ├── pg/            # Parameter Groups — система хранения конфигурации
│   ├── scheduler/     # Кооперативный планировщик задач
│   ├── config/        # Загрузка/сохранение конфигурации в EEPROM/Flash
│   ├── common/        # Утилиты: математика, фильтры, цвета, оси
│   ├── build/         # Дебаг-макросы, версионирование
│   ├── target/        # Конфигурации плат (пинмап, периферия, фичи)
│   └── main.c         # Точка входа
│
├── src/platform/      # MCU-специфичный код (STM32, AT32, GD32, APM32, ESP32, PICO)
├── src/test/          # Модульные тесты (Google Test)
├── lib/main/          # Сторонние библиотеки (CMSIS, HAL, MAVLink, DroneCAN)
├── mk/                # Makefile-фрагменты сборки
└── Makefile           # Точка входа сборки
```

## 3. Entrypoints

### Сборка
- **`Makefile`** (корень проекта) — точка входа. Выбор цели: `make TARGET=<name>`
- **`mk/source.mk`** — список всех исходников
- **`mk/config.mk`** — флаги компиляции, опции

### Прошивка
- **`src/main/main.c`** — `main()` → инициализация → `run()` → бесконечный цикл `scheduler()`
- **`src/main/fc/init.c`** — `initPhase1()`, `initPhase2()`, `initPhase3()` — полная последовательность инициализации железа, драйверов, PG, задач
- **`src/main/fc/core.c`** — `scheduler()` — диспетчеризация задач реального времени
- **`src/main/fc/tasks.c`** — регистрация всех задач (PID, RX, telemetry, blackbox, OSD и т.д.)

## 4. Основные модули

| Модуль | Директория | Назначение |
|--------|-----------|------------|
| **Flight Control** | `fc/` | Инициализация, главный цикл, RC-контролы, режимы полёта, задачи |
| **PID Controller** | `flight/` | PID-регулятор, фильтрация, микшер моторов/серв |
| **Sensors** | `sensors/` | Gyro, accel, baro, mag, GPS — инициализация и чтение |
| **Drivers (HAL)** | `drivers/` | Абстракция периферии: SPI, I2C, UART, ADC, DMA, таймеры, PWM, DShot |

| **Telemetry** | `telemetry/` | CRSF, SmartPort, HoTT, MAVLink, IBUS |
| **Blackbox** | `blackbox/` | Логирование полётных данных на Flash/SD |
| **OSD** | `osd/` | Наложение видеоинформации (MAX7456) |
| **CMS** | `cms/` | Экранное меню настройки |
| **MSP** | `msp/` | Протокол связи с Betaflight App |
| **CLI** | `cli/` | Терминальная конфигурация |
| **PG** | `pg/` | Parameter Groups — персистентная конфигурация |
| **Scheduler** | `scheduler/` | Кооперативный планировщик реального времени |
| **Config** | `config/` | Сохранение/загрузка конфигурации |
| **Target** | `target/` | Конфигурации конкретных плат |
| **Platform** | `src/platform/` | MCU-специфичная реализация (STM32, AT32 и т.д.) |

## 5. Порядок чтения кода (рекомендуемый маршрут)

Для понимания проекта **сверху вниз**:

1. **`main.c`** — точка входа, общий поток: `main()` → `initPhase1/2/3()` → `run()` → `scheduler()`
2. **`fc/init.c`** — полная картина инициализации: что и в каком порядке стартует
3. **`fc/core.c`** — главный цикл, работа планировщика
4. **`fc/tasks.c`** — какие задачи существуют и с какой частотой
5. **`flight/pid.c`** + **`flight/mixer.c`** — сердце управления полётом
6. **`sensors/gyro.c`** — чтение гироскопа, основа PID-цикла
7. **`rx/`** — как приходят команды с пульта
8. **`pg/`** — как хранятся настройки
9. **`drivers/`** — HAL, если нужно углубиться в железо
10. **`target/`** — конфигурации конкретных плат
11. **`msp/`** + **`cli/`** — интерфейсы взаимодействия с пользователем
12. **`blackbox/`** + **`osd/`** + **`telemetry/`** — периферийные фичи

# Отчёт об AI-инструменте: Cline (VS Code Extension)

## Общая информация

| Параметр | Значение |
|----------|----------|
| Инструмент | Cline (ранее Claude Dev) |
| Тип | IDE-плагин для Visual Studio Code |
| Базовый LLM | Anthropic Claude (Sonnet, Haiku, Opus) |
| Официальный сайт | https://cline.bot |
| Документация | https://docs.cline.bot |
| Репозиторий | https://github.com/cline/cline |

## Как подключаются инструкции к проекту

Cline использует механизм **`.clinerules/` directory** для подключения правил и инструкций к проекту:

### Формат и расположение
- Директория `.clinerules/` в корне проекта
- Один или несколько Markdown-файлов (`.md`) внутри этой директории
- Каждый файл содержит правила для определённого аспекта работы агента

### В нашем проекте используются следующие файлы `.clinerules/`:

| Файл | Назначение |
|------|-----------|
| `.clinerules/coding-style.md` | Правила стиля C-кода (1TBS, 4 пробела, naming, typedef и т.д.) |
| `.clinerules/project-guide.md` | Главный файл инструкций: контекст проекта, требования к результату, примеры задач, проверочные требования |
| `.clinerules/default-rules.md` | Правила Memory Bank: структура документации, обновление файлов при изменениях |
| `.clinerules/update_prompts.md` | Правила документирования: формат CHANGES.md и PROMPTS.md |

### Приоритизация и применение правил

1. **`.clineignore`** — исключает файлы/директории из контекста агента (аналог `.gitignore`)
2. **`.clinerules/`** — правила автоматически загружаются в системный промпт агента при старте сессии
3. **`memory-bank/`** — дополнительный контекст, который агент читает и поддерживает
4. **`docs/ai/`** — дополнительные документы, на которые агент может ссылаться

Правила применяются **автоматически**: при каждом запросе агент получает содержимое `.clinerules/` как часть системного промпта и обязан им следовать.

### Memory Bank
Cline использует концепцию **Memory Bank** — набора файлов в `memory-bank/`, которые агент читает при старте сессии и обновляет после изменений:
- `projectbrief.md` — основание проекта
- `productContext.md` — контекст и проблемы
- `activeContext.md` — текущий фокус и изменения
- `systemPatterns.md` — архитектура и паттерны
- `techContext.md` — технологии и инструменты
- `progress.md` — прогресс и известные проблемы

## Сравнение с альтернативами

| Инструмент | Механизм инструкций | Формат |
|-----------|-------------------|--------|
| Cline | `.clinerules/` directory | Markdown (любые .md) |
| GitHub Copilot | `.github/copilot-instructions.md` | Markdown |
| Cursor | `.cursorrules` | Markdown |
| Claude Code | `CLAUDE.md` | Markdown |
| Aider | `AGENTS.md` / `AI_GUIDE.md` | Markdown |

## Вывод
Cline предоставляет гибкий механизм `.clinerules/` для подключения контекстных инструкций. Файлы в этой директории автоматически загружаются в промпт агента, что обеспечивает принудительное и единообразное следование заданным правилам.

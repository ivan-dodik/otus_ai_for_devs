# Definition of Done — Чеклист завершения задачи

Каждый пункт ДОЛЖЕН быть выполнен перед объявлением задачи завершённой.

## Планирование

- [ ] Составлен план действий с перечнем изменяемых файлов
- [ ] План утверждён (если требуется)

## Код

- [ ] Код написан в соответствии с `.clinerules/coding-style.md`
  - [ ] 1TBS, 4 пробела, без табов
  - [ ] Правильная расстановка скобок (braces required)
  - [ ] Длина строк ≤ 120 (макс 140)
  - [ ] lowerCamelCase для функций/переменных
  - [ ] Правильные typedef (enum с/без count, struct с именем)
  - [ ] const-correctness соблюдена
  - [ ] Нет множественных statement/assignments на одной строке
  - [ ] Нет continue/goto/multiple return (без веской причины)
- [ ] Новые файлы содержат преамбулу с описанием назначения
- [ ] Нет неиспользуемых includes
- [ ] `#pragma once` используется вместо include guards

## Архитектура

- [ ] Нет upwards calls (драйверы не вызывают код из верхних слоёв)
- [ ] Target-specific код только в `src/main/target/`
- [ ] Нет нарушений существующих публичных API
- [ ] Нет изменений в конфигурационных структурах без необходимости

## Сборка и тесты

- [ ] `make` компилируется без ошибок (или `make <target>` для конкретной платформы)
- [ ] Существующие тесты проходят (если применимо)
- [ ] Для новой логики написаны unit-тесты (если применимо)

## Документирование

- [ ] `CHANGES.md` обновлён (дата, описание, список затронутых файлов)
- [ ] `PROMPTS.md` обновлён (добавлена запись о промпте)
- [ ] `docs/ai/PLAN.md` обновлён (отмечен прогресс)
- [ ] `memory-bank/activeContext.md` обновлён (если изменения существенные)

## Проверка

- [ ] Нет регрессий (существующий функционал не сломан)
- [ ] Изменения проверены на соответствие инструкциям проекта
- [ ] Если задача требовала конфигурации — новые параметры добавлены в `src/main/pg/`

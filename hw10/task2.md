# Домашнее задание: Дополнение проекта инструкциями для ИИ

## Цель

Научиться готовить проект к работе с AI-инструментами так, чтобы:

* модель быстрее входила в контекст;
* действовала в заданных границах;
* выдавала воспроизводимый результат.

## Описание/Пошаговая инструкция выполнения домашнего задания

В результате выполнения этого домашнего задания в репозитории появляется набор AI-инструкций, который:

* описывает контекст и архитектуру проекта;
* задаёт правила работы и внесения изменений;
* содержит примеры типовых задач и промптов;
* позволяет проверить, что ИИ действительно следует этим инструкциям.

### Что нужно сделать

Вы берёте учебный или пет-проект и добавляете в него инструкции для ИИ так, чтобы любой инженер и любой AI-агент могли одинаково корректно работать с кодовой базой. Можно использовать ИИ для генерации инструкций.

## Требования к выбору проекта

**Можно использовать:**

* Проект из предыдущего ДЗ
* Любой ваш публичный репозиторий, если там есть реальная логика
* Форк опенсорса, если вы вносите изменения в своём форке

**Ограничения:**

* нельзя использовать закрытые данные, секреты или конфиденциальный код;
* проект должен быть не слишком простым: несколько файлов/модулей, есть бизнес‑логика, есть точки расширения.

## Какие файлы / артефакты нужно добавить

**Важно:** Перед добавлением инструкций обязательно ознакомьтесь с официальной документацией вашего AI-инструмента (IDE‑плагин, CLI‑агент, standalone‑приложение) о том:

* как именно подключаются правила и инструкции к проекту;
* какие форматы файлов поддерживаются;
* как эти инструкции приоритизируются и применяются агентом.

Инструкции в проекте должны быть добавлены в соответствии с правилами выбранного инструмента, и AI‑агент при проверке обязан им следовать автоматически, если это поддерживается.

### Примеры механизмов и файлов по инструментам (ориентиры)

Ниже приведены примеры того, как инструкции подключаются в разных AI‑инструментах. Это не требования, а ориентиры: вы должны использовать тот механизм, который официально поддерживает ваш инструмент.

#### Claude (Claude Code, Claude Projects)

* `CLAUDE.md` — основной файл с правилами и контекстом проекта;
* описание архитектуры, ограничений и формата ответов;
* примеры типовых задач и ожидаемого поведения агента.

#### GitHub Copilot (Copilot Chat / Copilot Workspace)

* [Custom instructions for GitHub Copilot](https://code.visualstudio.com/docs/copilot/customization/custom-instructions#_use-instructionsmd-files)
* [Add repository instructions](https://docs.github.com/en/copilot/how-tos/configure-custom-instructions/add-repository-instructions?tool=vscode)
* `copilot-instructions.md` — основной файл инструкций репозитория;
* правила код-стайла, тестирования и допустимых изменений;
* указание, какие части проекта можно и нельзя модифицировать.

#### Cursor

* [Cursor Rules Documentation](https://cursor.com/docs/context/rules)
* `.cursorrules` — основной файл правил для агента;
* дополнительные документы (`docs/ai/*.md`) как источник контекста;
* явные указания по формату ответа, шагам изменений и допустимым операциям;
* правила, которые должны автоматически применяться при каждом запросе.

#### Aider

* `AGENTS.md` или `AI_GUIDE.md` — entrypoint для агента;
* ссылки на документы контекста и архитектуры;
* правила коммитов, ограничения и Definition of Done.

#### Универсальный вариант (если инструмент не навязывает формат)

* `docs/ai/CONTEXT.md` — контекст проекта;
* `docs/ai/TASKS.md` — типовые задачи и промпты;
* `docs/ai/STYLEGUIDE.md` — стиль и требования;
* `docs/ai/ARCHITECTURE.md`, `DECISIONS.md`, `KNOWN_ISSUES.md`, `DEFINITION_OF_DONE.md` — по необходимости.

## Содержание AI-инструкций

Ваши инструкции должны включать следующие блоки.

**Примечание:** некоторые инструменты уже преднастроены к определённому формату работы (например, всегда делают план действий сначала). В таком случае эту часть можно опустить.

### 1. Контекст и границы

* назначение проекта (1–2 абзаца);
* основные модули/папки и их ответственность;
* ключевые сущности и термины (мини-глоссарий);
* ограничения: что нельзя менять и почему;
* должен ли ИИ задавать уточняющие вопросы или может делать допущения.

### 2. Требования к результату

* стиль кода и форматирование;
* требования к тестам и проверкам;
* формат ответа ИИ:
  * план действий;
  * изменения в коде;
  * пояснение решений;
  * шаги проверки.

### 3. Примеры типовых задач

**Минимум 3 сценария**, например:

* добавление новой функции или endpoint;
* покрытие модуля тестами;
* исправление бага по описанию;
* рефакторинг без изменения поведения.

Для каждого сценария:

* пример промпта;
* ожидаемый формат ответа;
* критерии готовности.

### 4. "Проверочные" требования (не обязательно)

Следование ИИ общим инструкциям может быть сложно отследить. Добавьте в инструкции "проверочные" требования для упрощения проверки. Выполнение таких инструкций должно быть легко отслеживаемо.

**Пример:** *"в начале каждого файла должна быть такая-то преамбула"* или даже новогодне-шуточное *"каждый комментарий должен оканчиваться фразой ХО ХО ХО!"*

## Пошаговая инструкция выполнения ДЗ

1. **Опишите проект “для новичка”**  
   Кратко: что делает проект, основные сценарии использования, как его запустить.

2. **Зафиксируйте структуру и границы**  
   Опишите модули, ответственность и ограничения на изменения.

3. **Сформулируйте правила взаимодействия с ИИ**  
   Как задавать задачи, какие правила и ограничения обязан соблюдать агент.

4. **Добавьте примеры типовых задач**  
   Минимум 3 сценария с промптами и ожидаемым результатом.

5. **Проверка инструкций через AI-агента (обязательный шаг)**  
   * Попросите AI-агента модифицировать проект (небольшая фича, багфикс или рефакторинг).  
   * Явно укажите, что агент обязан строго следовать добавленным AI-инструкциям.  
   * Проверьте и зафиксируйте:  
     * соблюдение правил и ограничений;  
     * соответствие формату ответа;  
     * отсутствие архитектурных нарушений.  
   * При необходимости доработайте инструкции.

6. **Сохраните все использованные промпты**  
   Все запросы к ИИ необходимо оформить и приложить списком в Markdown.

## Формат сдачи

* Ссылка на PR
* **Отчёт:** Краткое описание AI-инструмента, для которого вы специализируете инструкции
* **Отчёт:** Краткое описание того, что сделано, и как прошла проверка инструкций:
  * опишите изменения, которые хотели, чтобы агент внёс;
  * промпт;
  * опишите результат, видно ли, что он чётко соответствует инструкциям проекта
* **Отчёт:** Раздел с описанием трудностей при выполнении ДЗ (не обязательно)
* **Раздел «Использованные промпты»** в формате Markdown

## Критерии оценки

Работа считается выполненной, если:

* Проект выбран корректно и содержит реальную логику.
* Описан выбранный AI-инструмент (в отчёте).
* Добавлены AI-документы в соответствии с правилами выбранного инструмента.
* Инструкции содержат контекст, правила, требования и примеры задач.
* Проведена практическая проверка через AI-агента, проверка описана в отчёте.
* Все использованные промпты оформлены в явном Markdown-виде.

# Product Context: OpenBCM Helper MCP Server

## Почему этот проект существует
Разработчики под Broadcom ASIC (Tomahawk, Trident, Jericho и др.) тратят значительное время на навигацию по огромному SDK. SDK содержит тысячи API, сотни заголовочных файлов, CINT скрипты для тестирования и chip-specific код. AI-ассистенты (Cline, Continue, OpenCode) могут помочь, но им нужен инструмент для эффективного поиска и анализа SDK.

## Какие проблемы решает
1. **Поиск API:** разработчик знает имя функции (`bcm_vlan_create`), но не помнит сигнатуру, параметры, в каком заголовочном файле
2. **Поиск примеров:** нужно найти, как используется API в существующем коде или CINT скриптах
3. **Трассировка реализации:** от публичного API до chip-specific кода (как работает dispatch)
4. **Информация о чипе:** какие модули поддерживаются, feature macros, dev ids
5. **CINT скрипты:** поиск тестовых скриптов по функционалу и чипу

## Как это работает
1. Пользователь задаёт вопрос агенту в IDE (например, "Найди API bcm_vlan_create")
2. Агент определяет, какой MCP tool вызвать
3. MCP-сервер получает запрос, выполняет поиск по SDK через ripgrep или индекс
4. Сервер возвращает структурированный JSON-ответ
5. Агент обрабатывает результат и отвечает пользователю

## Пользовательский опыт
- **Быстро:** ответ за секунды (благодаря индексации и ripgrep)
- **Структурированно:** JSON с полями, а не сырой текст
- **Безопасно:** read-only, sandbox, никаких изменений в SDK
- **Контекстно:** для CINT скриптов извлекаются заголовочные комментарии с описанием

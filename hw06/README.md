# Ревью кода Betaflight — обзор выполненной работы

## Цель

Проведён анализ кода opensource-проекта [Betaflight](https://github.com/betaflight/betaflight) (прошивка для полётных контроллеров БПЛА) с использованием AI-ассистента. Результат оформлен в формате, близком к профессиональному code review.

## Область анализа

- Обработка сигналов от пульта (RC): от приёма радиосигнала до подачи в PID-регулятор и на двигатели
- Логика ARM/DISARM: механизмы включения/выключения полётного режима (стики, переключатель, failsafe, crashflip, runaway takeoff)
- Выявление логических ошибок, уязвимостей безопасности, нарушений best practices и антипаттернов

## Результаты

- **29 проблем** (2 High, 17 Medium, 10 Low) по 4 категориям
- **Детальная карта зависимостей** прохождения RC сигналов с примерами кода
- **12 предложений улучшений** (срочные, среднесрочные, долгосрочные)
- **Полный список промптов**, использованных в ходе анализа

## Ключевые находки

1. **Race condition в ARM/DISARM логике** — статические переменные и переключение между стиками и переключателем могут привести к непреднамеренному ARM/DISARM в полёте
2. **30+ magic numbers в rc_adjustments.c** — лимиты PID-параметров не синхронизированы с cli.c
3. **DISARM через стики не проверяет BOXSTICKCOMMANDDISABLE** — несоответствие с ARM
4. **Debounce DISARM всего 3 тика (~375 мкс)** — недостаточная защита от ложных срабатываний
5. **God-функция processRcStickPositions()** — 10 различных задач в одной функции
